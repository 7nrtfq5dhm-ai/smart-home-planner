ERD — Проектирование базы данных

Обоснование ключевых решений​

user → project (1:N)​

Один пользователь создаёт много проектов, но каждый план принадлежит строго одному автору. Внешний ключ user_id хранится в таблице project.

Все ID используют тип UUID вместо SERIAL. Если бы ID был числом 123, злоумышленник мог бы подставить 124 и украсть чужой проект. UUID угадать невозможно.

project ↔ asset через project_asset (M:N, Паттерн L1)​

Один стул из каталога может стоять в сотнях проектов. В одном проекте — десятки разных стульев. Промежуточная таблица project_asset хранит не только связь, но и координаты pos_x, pos_y и слой (мебель/электрика).

asset_category — иерархия каталога (петля 1:N, Паттерн L4)​

Каталог — это дерево: Инженерные системы → Водоснабжение → Трубы → ПВХ трубы. Поле parent_id ссылается на ту же таблицу — можно создавать любую глубину вложенности без изменения схемы БД.

project → ai_task (1:N)​

Один проект содержит множество AI-запросов («сделай кухню больше», «добавь розетки»). История хранится для очереди RabbitMQ и аналитики.

project → project_canvas_data (1:1, Паттерн P1)​

В project хранится лёгкая мета-информация (название, дата, автор) — дашборд загружается мгновенно. Тяжёлый чертёж (стены, комнаты — несколько МБ) вынесен в project_canvas_data и загружается только при открытии редактора. Формат хранения — JSONB, так как структура плана гибко меняется.

Схема таблиц​

┌──────────────┐         ┌─────────────────────┐
│     user     │         │       project        │
│──────────────│         │─────────────────────│
│ id (UUID) PK │──1:N───►│ id (UUID) PK        │
│ email        │         │ user_id (FK → user) │
│ password_hash│         │ name                │
│ subscription │         │ status_id (FK)      │
│ created_at   │         │ created_at          │
└──────────────┘         │ updated_at          │
                         └─────────────────────┘
                                  │          │
                                 1:1        1:N
                                  │          │
                    ┌─────────────┘    ┌─────────────┐
                    │                  │   ai_task   │
          ┌─────────────────┐          │─────────────│
          │project_canvas_  │          │ id (UUID)   │
          │data             │          │ project_id  │
          │─────────────────│          │ prompt      │
          │ project_id PK,FK│          │ action_type │
          │ canvas_json JSONB│         │ status      │
          │ updated_at      │          │ created_at  │
          └─────────────────┘          └─────────────┘

┌──────────────────┐     ┌───────────────────┐     ┌─────────────┐
│  project_asset   │     │      asset        │     │asset_category│
│──────────────────│     │───────────────────│     │─────────────│
│ id (UUID) PK     │     │ id (UUID) PK      │     │ id (UUID) PK│
│ project_id (FK)  │────►│ name              │◄────│ name        │
│ asset_id (FK)    │     │ category_id (FK)  │     │ parent_id   │◄─┐
│ pos_x (FLOAT)    │     │ model_2d (JSONB)  │     └─────────────┘  │
│ pos_y (FLOAT)    │     └───────────────────┘           │           │
│ layer (VARCHAR)  │                                      └───────────┘
└──────────────────┘                                    (петля, nullable)
