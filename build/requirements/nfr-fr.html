Функциональные и нефункциональные требования

Шаблон

Данный раздел содержит шаблон для описания требований. Детальное заполнение — следующий этап работы над документацией.

Функциональные требования (FR)​

Функциональные требования описывают что система должна делать.

FR-01: Управление пользователями​

IDТребование
FR-01.1Система должна позволять регистрацию через email или номер телефона с OTP-подтверждением
FR-01.2Система должна поддерживать авторизацию через JWT-токены
FR-01.3Система должна блокировать вход на 15 минут после 5 неудачных попыток

FR-02: Управление проектами​

IDТребование
FR-02.1Система должна позволять создавать проект с нуля с заданием габаритов помещения
FR-02.2Система должна позволять генерацию проекта через текстовый запрос к AI
FR-02.3Система должна ограничивать количество проектов для Free-аккаунта (не более 3)
FR-02.4Система должна поддерживать автосохранение проекта

FR-03: Редактор​

IDТребование
FR-03.1Редактор должен отображать план помещения на интерактивном холсте
FR-03.2Редактор должен поддерживать работу со слоями (мебель, электрика, водоснабжение, канализация)
FR-03.3Редактор должен позволять расстановку объектов из каталога

FR-04: AI-помощник​

IDТребование
FR-04.1Система должна принимать текстовые команды для изменения планировки
FR-04.2Для Free-аккаунта счётчик AI-запросов должен сбрасываться ежедневно в 00:00 UTC
FR-04.3В MVP AI-команды не изменяют инженерные коммуникации

Нефункциональные требования (NFR)​

Нефункциональные требования описывают как система должна работать.

Производительность​

IDТребование
NFR-01.1Дашборд со списком проектов должен загружаться не более чем за 1 секунду
NFR-01.2REST API должен отвечать на 95% запросов не более чем за 300 мс
NFR-01.3[Указать требования к времени генерации AI]

Надёжность​

IDТребование
NFR-02.1Потеря задачи на генерацию AI недопустима: RabbitMQ должен повторить задачу при падении сервиса
NFR-02.2[Указать SLA/uptime]

Безопасность​

IDТребование
NFR-03.1Все ID ресурсов должны быть UUID для предотвращения перебора
NFR-03.2Платёжные данные пользователей не хранятся на серверах системы (обработка через ЮKassa)
NFR-03.3[Указать требования к шифрованию]

Масштабируемость​

IDТребование
NFR-04.1AI-микросервис должен горизонтально масштабироваться за счёт добавления Consumer-воркеров
NFR-04.2[Указать ожидаемую нагрузку]
