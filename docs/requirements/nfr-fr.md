# Функциональные и нефункциональные требования

## Функциональные требования (FR)

### FR-01 — Управление пользователями

| ID | Требование |
|----|-----------|
| FR-01.1 | Регистрация через email или телефон с OTP-подтверждением |
| FR-01.2 | Авторизация через JWT-токены |
| FR-01.3 | Блокировка входа на 15 минут после 5 неудачных попыток |

### FR-02 — Управление проектами

| ID | Требование |
|----|-----------|
| FR-02.1 | Создание проекта с заданием габаритов помещения |
| FR-02.2 | Генерация проекта через текстовый запрос к AI |
| FR-02.3 | Ограничение: не более 3 активных проектов для Free-аккаунта |
| FR-02.4 | Автосохранение проекта при изменениях |

### FR-03 — Редактор

| ID | Требование |
|----|-----------|
| FR-03.1 | Интерактивный холст для работы с планом помещения |
| FR-03.2 | Поддержка слоёв: мебель, электрика, водоснабжение, канализация |
| FR-03.3 | Расстановка объектов из каталога (мебель, техника, розетки) |

### FR-04 — AI-помощник

| ID | Требование |
|----|-----------|
| FR-04.1 | Приём текстовых команд для изменения планировки |
| FR-04.2 | Сброс счётчика AI-запросов для Free-аккаунта в 00:00 UTC |
| FR-04.3 | В MVP AI-команды не изменяют инженерные коммуникации |

---

## Нефункциональные требования (NFR)

### Производительность

| ID | Требование |
|----|-----------|
| NFR-01.1 | Дашборд со списком проектов загружается за < 1 секунды |
| NFR-01.2 | REST API отвечает на 95% запросов за < 300 мс |

### Надёжность

| ID | Требование |
|----|-----------|
| NFR-02.1 | RabbitMQ повторяет AI-задачу при падении сервиса (Acknowledgment) |
| NFR-02.2 | SLA системы: 99.9% uptime |

### Безопасность

| ID | Требование |
|----|-----------|
| NFR-03.1 | Все ID ресурсов — UUID (защита от перебора чужих данных) |
| NFR-03.2 | Платёжные данные не хранятся на серверах системы (обработка через ЮKassa) |

### Масштабируемость

| ID | Требование |
|----|-----------|
| NFR-04.1 | AI-микросервис масштабируется горизонтально добавлением Consumer-воркеров |
