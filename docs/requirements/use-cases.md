# Use Cases

## UC-01 — Регистрация нового пользователя

| Поле | Значение |
|------|---------|
| **Акторы** | Пользователь, Email/SMS-сервис, Backend API |
| **Предусловия** | Пользователь открыл сайт, не авторизован |
| **Постусловия (успех)** | Аккаунт создан, пользователь авторизован, статус подписки = Free |
| **Постусловия (неудача)** | Аккаунт не создан, отображается сообщение об ошибке |

### Основной поток

| № | Действие пользователя | Отклик системы |
|---|----------------------|----------------|
| 1 | Нажимает «Зарегистрироваться» | Форма ввода email или телефона |
| 2 | Вводит email или телефон | Валидация формата |
| 3 | — | Отправка OTP-кода |
| 4 | Вводит OTP-код | Проверка кода |
| 5 | Вводит и повторяет пароль | Проверка совпадения |
| 6 | — | Создание аккаунта в БД, выдача JWT |
| 7 | — | Редирект на главный экран |

### Альтернативные потоки

**Неверный OTP-код:** сообщение «Код неверный или истёк», возможность запросить повторно.

**Email уже зарегистрирован:** сообщение «Аккаунт уже существует», Use Case завершается.

**Пароли не совпадают:** сообщение «Пароли не совпадают», повторный ввод.

### Бизнес-правила

- Email или телефон должны быть уникальными
- OTP-код действует 5 минут
- Минимальная длина пароля — 8 символов

## UC-02 — Авторизация пользователя

| Поле | Значение |
|------|---------|
| **Акторы** | Зарегистрированный пользователь, Backend API |
| **Постусловия (успех)** | Активная сессия (JWT), пользователь на главном экране |
| **Постусловия (неудача)** | Сессия не создана, отображается ошибка |

### Основной поток

1. Пользователь вводит email/телефон и пароль
2. Backend проверяет учётные данные
3. При успехе создаётся JWT-токен
4. Редирект на главный экран

### Бизнес-правила

- После 5 неудачных попыток — блокировка на 15 минут

## UC-03 — Создание проекта с нуля

| Поле | Значение |
|------|---------|
| **Предусловия** | Пользователь авторизован, лимит проектов не исчерпан |
| **Постусловия (успех)** | Проект создан, открыт редактор |

### Основной поток

| № | Действие пользователя | Отклик системы |
|---|----------------------|----------------|
| 1 | Нажимает «Создать проект» | Модальное окно: название, ширина × длина |
| 2 | Вводит параметры | Валидация |
| 3 | Нажимает «Создать» | Запись в БД, открытие редактора |

### Бизнес-правила

- Габариты > 0 и ≤ 100 м
- Для Free-аккаунта — не более 3 активных проектов

## UC-04 — Редактирование через AI-команду

| Поле | Значение |
|------|---------|
| **Предусловия** | Проект открыт в редакторе |
| **Постусловия (успех)** | Изменения применены, холст перерисован |

### Основной поток

1. Пользователь вводит текстовую команду («Сделай кухню больше»)
2. Фронтенд отправляет команду + состояние проекта на Backend
3. AI-модуль генерирует изменения
4. Редактор перерисовывает план
5. Уведомление «Изменения применены»

### Бизнес-правила

- Для Free-подписки счётчик AI-запросов сбрасывается в 00:00 UTC
- В MVP AI-команды не изменяют инженерные коммуникации
