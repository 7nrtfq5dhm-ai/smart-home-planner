# Технологии хранения данных

## Сущности и паттерны взаимодействия

| Сущность | Атрибуты | Акторы | Характер |
|---------|----------|--------|---------|
| **Пользователь** (User) | ID, Email, Хеш пароля, Тип подписки | Пользователь, Система аутентификации | OLTP: высокая консистентность, безопасность |
| **Метаданные проекта** (Project) | ID, ID владельца, Название, Дата, Статус | Пользователь, Backend | OLTP: частое чтение списка проектов |
| **План помещения** (Layout Data) | Стены, Комнаты, Объекты, Инженерные узлы (JSON) | Пользователь (редактор), AI-сервис | OLTP: большой объём, гибкая структура |
| **Задача на генерацию** (AI Task) | TaskID, ProjectID, Промпт, Статус, Логи | Backend, AI-сервис, Брокер | OLTP: высокая запись, короткий TTL |
| **Библиотека объектов** (Asset) | ID, Категория, 2D/3D модель, Метаданные | Пользователь (каталог) | Справочник: редкие изменения, частое чтение |
| **Аналитика** (Usage Analytics) | ID действия, Время, Инструменты, Ошибки | Аналитик, Система | OLAP: огромные объёмы, периодическое чтение |

## Выбор технологий хранения

| Сущность | Требования | Технология | Обоснование |
|---------|-----------|------------|-------------|
| Пользователи и метаданные проектов | Сложные связи (FK), строгая целостность (ACID) | **PostgreSQL** | Надёжное хранение связей «один пользователь — много проектов» |
| План помещения (чертёж) | Большой объём, меняющаяся структура | **MongoDB** или **PostgreSQL JSONB** | План — сложный вложенный объект; хранить в строках таблиц неудобно |
| Задачи ИИ (очередь статусов) | Очень быстрый доступ, не вечное хранение | **Redis** | Идеально для статусов задач в реальном времени и кэширования |
| Файлы объектов (модели, текстуры) | Огромные бинарные данные | **Object Storage (S3)** | БД плохо работают с тяжёлыми файлами; S3 — стандарт для статики |
| Аналитика | Петабайты данных, потоковая запись | **Data Lake / Columnar DB** | Аналитические запросы требуют колоночного хранения |

## Схема интерфейса (экраны → API)

| Экран | Действие | HTTP-метод и Endpoint | Описание |
|-------|---------|----------------------|---------|
| `/` Дашборд | Загрузка проектов | `GET /api/v1/projects` | Массив метаданных проектов пользователя |
| `/generate` Генерация | Создание через AI | `POST /api/v1/projects/generate` | Текст промпта → ID нового проекта |
| `/editor/{projectId}` | Загрузка плана | `GET /api/v1/projects/{projectId}` | Данные конкретного проекта для холста |
| `/editor/{projectId}` | AI-команда | `POST /api/v1/projects/{projectId}/ai-commands` | Текстовая команда для точечного изменения |
| `/editor/{projectId}` | Сохранение | `PUT /api/v1/projects/{projectId}` | Обновление данных проекта в БД |
